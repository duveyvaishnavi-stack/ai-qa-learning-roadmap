## Week 1 — Python + Anthropic API

> Goal: Write your first AI script that calls Claude and gets a response

### What you'll learn
- Python basics — variables, functions, loops, JSON
- How to call the Anthropic API
- How to send a prompt and read the response
- How to structure output as JSON

### Resources
| Resource | Link | Time |
|----------|------|------|
| Python for Everybody (Coursera) | [coursera.org/specializations/python](https://coursera.org/specializations/python) | 3–4 hrs |
| Anthropic API Quickstart | [docs.anthropic.com/en/docs/quickstart](https://docs.anthropic.com/en/docs/quickstart) | 30 mins |
| Anthropic Python SDK | [github.com/anthropics/anthropic-sdk-python](https://github.com/anthropics/anthropic-sdk-python) | 20 mins |

### Week 1 project 🛠️
Run `orchestrator.py` from this repo:
```bash
pip install anthropic
export ANTHROPIC_API_KEY=your-key
python orchestrator.py
```
You'll get a real Playwright `.spec.ts` file generated by AI. Done!

### Key concepts
- `client.messages.create()` — how to call Claude
- `model` — which Claude model to use (`claude-sonnet-4-20250514`)
- `max_tokens` — how long the response can be
- `messages` — the conversation history you send

## Week 2 — Prompt Engineering

> Goal: Get better, more consistent output from Claude

### What you'll learn
- How to write clear, specific prompts
- System prompts vs user prompts
- Chain of thought reasoning
- XML tags for structured output
- How to ask Claude to return JSON reliably

### Resources
| Resource | Link | Time |
|----------|------|------|
| Anthropic Prompt Engineering Guide | [docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview) | 2 hrs |
| DeepLearning.AI — Prompt Engineering | [deeplearning.ai/short-courses/chatgpt-prompt-engineering-for-developers](https://deeplearning.ai/short-courses/chatgpt-prompt-engineering-for-developers) | 2 hrs |
| Anthropic Prompt Library | [docs.anthropic.com/en/prompt-library](https://docs.anthropic.com/en/prompt-library) | Browse |

### Week 2 project 🛠️
Improve the prompt in `orchestrator.py`:
- Add a system prompt that sets Claude's role as a senior QA engineer
- Ask Claude to think step by step before generating test cases
- Use XML tags to separate sections of your prompt
- Compare the output quality before and after

### Key concepts
- **Be specific** — vague prompts = vague output
- **Give examples** — show Claude what good output looks like
- **Use XML tags** — `<story>`, `<standards>`, `<output_format>`
- **Ask for JSON** — always say "Return ONLY a JSON array, no other text"

## Week 3 — RAG (Retrieval Augmented Generation)

> Goal: Give Claude your private knowledge — codebase, standards, past incidents

### What you'll learn
- What embeddings are and how they work
- How vector databases store and search by meaning
- How to embed your documents
- How to query a vector DB and inject results into your prompt

### Resources
| Resource | Link | Time |
|----------|------|------|
| DeepLearning.AI — Building RAG | [deeplearning.ai/short-courses/building-evaluating-advanced-rag](https://deeplearning.ai/short-courses/building-evaluating-advanced-rag) | 3 hrs |
| pgvector (free vector DB in Postgres) | [github.com/pgvector/pgvector](https://github.com/pgvector/pgvector) | Setup: 1 hr |
| OpenAI Embeddings (works with any text) | [platform.openai.com/docs/guides/embeddings](https://platform.openai.com/docs/guides/embeddings) | 1 hr |
| Chroma (easy local vector DB) | [docs.trychroma.com](https://docs.trychroma.com) | 1 hr |

### Week 3 project 🛠️
Add a RAG layer to `orchestrator.py`:
1. Take your Playwright page objects and embed them
2. Store in pgvector or Chroma
3. When a story comes in — search for relevant page objects
4. Inject them into the Claude prompt as context
5. Watch the generated tests use YOUR actual selectors

### Key concepts
- **Embedding** — converting text into a list of numbers (vectors)
- **Similarity search** — finding chunks that are mathematically close
- **Chunking** — splitting documents into ~500 word passages
- **Context injection** — adding retrieved chunks to your Claude prompt

## Week 4 — MCP (Model Context Protocol)

> Goal: Connect Claude to real tools — Jira, GitHub, TestRail

### What you'll learn
- What MCP is and how it works
- How to use pre-built MCP servers
- How Claude calls tools and reads results
- How to build a simple MCP connector

### Resources
| Resource | Link | Time |
|----------|------|------|
| MCP Official Docs | [modelcontextprotocol.io](https://modelcontextprotocol.io) | 2 hrs |
| MCP Servers Directory | [github.com/modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers) | Browse |
| Anthropic MCP Guide | [docs.anthropic.com/en/docs/agents-and-tools/mcp](https://docs.anthropic.com/en/docs/agents-and-tools/mcp) | 1 hr |

### Week 4 project 🛠️
Add the Jira MCP connector (Stage 2 of this repo):
- Install the Jira MCP server
- Configure it with your Jira credentials
- Update `orchestrator.py` to fetch stories from Jira automatically
- No more copy-pasting stories into the script!

### Key concepts
- **MCP server** — a small process that exposes tools Claude can call
- **Tool call** — Claude decides to call a tool and passes parameters
- **Tool result** — the MCP server runs the action and returns data
- **Pre-built servers** — Jira, GitHub, Slack, Postgres already exist — use them!

## Week 5+ — AI Agents + Full Orchestration

> Goal: Build a fully automated pipeline that runs without you

### What you'll learn
- What AI agents are (AI in a loop, calling tools until done)
- How to handle multi-step workflows
- Error handling and retries
- Scheduling — run nightly or on webhook triggers
- Monitoring and logging your pipeline

### Resources
| Resource | Link | Time |
|----------|------|------|
| DeepLearning.AI — AI Agents in LangGraph | [deeplearning.ai/short-courses/ai-agents-in-langgraph](https://deeplearning.ai/short-courses/ai-agents-in-langgraph) | 4 hrs |
| Anthropic Agents Guide | [docs.anthropic.com/en/docs/agents-and-tools/tool-use](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use) | 2 hrs |
| GitHub Actions (for scheduling) | [docs.github.com/en/actions](https://docs.github.com/en/actions) | 1 hr |

### Week 5 project 🛠️
Complete the full pipeline:
```
Story moves to "Ready for QA" in Jira
        ↓ (Jira webhook triggers the pipeline)
AI reads story + fetches RAG context
        ↓
Generates test cases + Playwright spec
        ↓
Opens GitHub PR automatically
        ↓
Posts summary to Slack
        ↓
Done — zero keyboard interaction
```

### Key concepts
- **Agentic loop** — Claude runs, calls tools, reads results, runs again
- **Webhook trigger** — Jira fires an event when a story changes status
- **GitHub Actions** — schedule your pipeline to run nightly
- **Observability** — log every step so you can debug when it fails
